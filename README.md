# Wiren Board Discovery for Home Assistant

Кастомная интеграция Home Assistant для автоматического поиска устройств Wiren Board по MQTT-топикам `/devices/...`.

## Установка через HACS

1. Откройте HACS.
2. Откройте `Custom repositories`.
3. Добавьте репозиторий:

`https://git.akinin.su/akininav/wiren-board-discovery`

4. Выберите тип `Integration`.
5. Установите `Wiren Board Discovery`.
6. Перезапустите Home Assistant.

## Что делает

- подключается к MQTT-брокеру Wiren Board;
- читает retained-топики устройств и контролов;
- показывает найденные контролы в мастере настройки Home Assistant;
- создает сущности `sensor`, `binary_sensor`, `switch`, `number`, `button` и `text`;
- скрывает системные устройства в списке выбора по умолчанию;
- проставляет классы и единицы измерения для основных сенсоров;
- отправляет команды записи в стандартный топик Wiren Board `/devices/<device>/controls/<control>/on`.

## Ручная установка

Скопируйте папку `custom_components/wirenboard_discovery` в `config/custom_components/` Home Assistant и перезапустите Home Assistant.

После перезапуска откройте:

`Настройки -> Устройства и службы -> Добавить интеграцию -> Wiren Board Discovery`

Адрес контроллера по умолчанию: `10.10.100.5`.

Параметры подключения можно изменить позже через `Настроить -> Подключение`: адрес, порт, логин, пароль и префикс MQTT-топиков.

Конфигурацию можно перенести между установками через `Настроить -> Экспорт конфигурации` и `Настроить -> Импорт конфигурации`.

## Совместимость с wb-engine

Интеграция не зависит от внутреннего формата `wb-engine`: она читает стандартное MQTT-дерево Wiren Board. Если `wb-engine` создает виртуальные устройства в Wiren Board, они тоже появятся в `/devices/...` и будут найдены.

## Группировка в устройства Home Assistant

По умолчанию одно устройство Wiren Board становится одним устройством Home Assistant.

Через интерфейс Home Assistant откройте интеграцию Wiren Board Discovery, нажмите `Настроить`, затем `Добавить или обновить группу`. Выберите название устройства и контролы, которые должны попасть в одно HA-устройство.

Группа может быть обычным HA-устройством или составной сущностью: ворота, шторы/роллеты, термостат, кондиционер. Для составных сущностей можно указать роли контролов: команда, позиция, состояние, препятствие, текущая и целевая температура, питание, режим, вентилятор. Поле `Иконка` принимает значения Home Assistant вида `mdi:gate`. Если контрол нужен и внутри составной сущности, и отдельно для автоматизаций, добавьте его в `Оставить отдельными сущностями`.

Также можно использовать файл `/config/wirenboard_discovery.yaml`:

```yaml
devices:
  kitchen_climate:
    name: "Кухня: климат"
    controls:
      - "wb-msw-v4_74/Current Temperature"
      - "wb-msw-v4_74/Humidity"
      - "R8C-963S_22/Power"
      - "R8C-963S_22/Temperature Setpoint"

  hallway_relays:
    name: "Прихожая: реле"
    controls:
      - "wb-mr6cu_63/K1"
      - "wb-mr6cu_63/K2"
      - "wb-mr6cu_63/K3"
```

В `controls` указывается ключ вида `<wb_device_id>/<control_id>`. Можно использовать `*`, например `"wb-mr6cu_63/K*"`. После изменения файла перезагрузите интеграцию или Home Assistant.

## Текущие ограничения

- Первый вариант покрывает базовые типы: датчики, бинарные датчики, выключатели, числовые `range`/writable `value`-контролы, кнопки и текстовые поля.
- Для `rgb`, `cover`, `climate` и расширенных классов устройств нужно добавить отдельные платформы после проверки реальных топиков на контроллере.
